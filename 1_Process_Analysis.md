# Process Analysis Document

**Candidate:** Vaibhav Singh
**Source:** Discovery session with Michael Reynolds (Managing Partner)

---

## 1. Process Summary

**Purpose:** Manage a consulting engagement end-to-end — from initial client contact through project delivery and billing — for the firm's client delivery practice.

**Key Stakeholders**
| Stakeholder | Role in Process |
|---|---|
| Michael Reynolds (Managing Partner) | Runs discovery calls, drafts most proposals, sometimes assigns work directly for high-priority clients |
| Consultants (incl. senior consultants) | Run some discovery calls and proposals, execute project work, submit timesheets |
| Project Manager | Creates Asana project + shared folders, typically assigns tasks based on availability/expertise |
| Accounting | Generates invoices from timesheets (hourly) or on a monthly schedule (fixed-fee) |
| Client | Requests engagement, approves/revises proposal, signs agreement, receives updates |

**Systems & Tools Referenced**
- HubSpot — lead logging (inconsistent)
- Asana — project tracking (primary, but not universally updated)
- Google Drive — one of several documentation locations
- Slack / Email — alternate channels for updates and client communication
- Spreadsheet — manual capacity/resource tracking

**Major Process Stages**
1. Client enquiry (referral / LinkedIn / website)
2. Discovery call
3. Proposal preparation and (possible) revision
4. Agreement signed
5. Kickoff meeting scheduled
6. Project setup (Asana + shared folders)
7. Resource allocation
8. Project execution
9. Progress tracking
10. Client communication
11. Billing
12. *(No confirmed formal closure step)*

**Inputs & Outputs**
| Inputs | Outputs |
|---|---|
| Client enquiry | Qualified lead |
| Discovery call notes | Proposal |
| Approved/revised proposal | Signed agreement |
| Signed agreement | Kickoff meeting, active project |
| Assigned consultants | Project deliverables |
| Timesheets | Invoice |

**Decision Points**
- Is the opportunity a good fit? (criteria not defined)
- Log the lead in HubSpot now, or skip it?
- Does the client approve the proposal, or request revisions?
- Is project setup complete before work starts, or does an urgent client justify starting early?
- Does the PM assign the consultant, or does Michael assign directly (high-priority override)?
- Are updates logged in Asana, or shared via Slack/email instead?
- Is the engagement fixed-fee (invoiced monthly) or hourly (invoiced from timesheets)?

## 2. Information Gaps

**Missing Information**
- Criteria for qualifying an opportunity as "a good fit"
- Any formal proposal review/approval step before it reaches the client (beyond the client's own review)
- What the kickoff meeting actually covers, and who's expected to attend
- A defined resource-planning methodology beyond a spreadsheet
- Any formal project closure or lessons-learned step
- A single documented location/standard for project files
- What happens when a project falls behind schedule — no escalation path was described
- How mid-project scope changes are handled

**Unclear Responsibilities**
- Who is accountable for a lead when it isn't logged in HubSpot — does anyone backfill it later?
- Who decides whether an engagement is billed fixed-fee vs. hourly?
- Who follows up when a consultant doesn't update Asana weekly?
- Who owns resolving a proposal stuck in multiple revision cycles?

**Assumptions Requiring Validation**
- Every engagement has a Project Manager assigned (not confirmed for smaller engagements)
- Asana and Google Drive are the *intended* systems of record, not just the most-used ones
- "Weekly" Asana updates are a hard expectation rather than a guideline
- The capacity spreadsheet is a single shared source, not per-person tracking

**Areas Requiring Additional Discovery**
- Lead qualification and proposal approval governance
- Resource planning methodology and ownership
- Project closure and knowledge-capture process
- Documentation storage policy
- Escalation procedure for at-risk projects
- Scope-change / change-management handling

## 3. Discovery Questions

**Workflow Clarification**
1. What criteria determine whether an opportunity is a good fit before a proposal is drafted?
2. What does the kickoff meeting typically cover, and who attends?
3. Under what specific conditions is a consultant allowed to start work before project setup is complete?

**Ownership & Responsibility**
4. Who, if anyone, reviews a proposal before it's sent when someone other than Michael drafts it?
5. Who is responsible for logging a lead in HubSpot when it's initially skipped — is there a backfill process?
6. Who decides which consultant is assigned when several are available and qualified?

**Business Rules**
7. What determines whether an engagement is billed fixed-fee or hourly?
8. Is there a deadline or escalation path for late timesheet submission?
9. Is weekly Asana updating a formal requirement, and is there a consequence for missing it?

**Process Exceptions**
10. What happens if a proposal goes through several rounds of revision without resolution?
11. Is there a defined escalation path when a project falls behind schedule?
12. How are scope changes requested mid-project handled?

**Documentation & Closure**
13. Is there a single required location for project documentation, or is the current mix (Asana/Drive/email) acceptable?
14. Is there any formal project closure step, or does the engagement simply end at final invoice?
